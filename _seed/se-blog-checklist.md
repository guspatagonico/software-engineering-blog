# SE Blog Post Checklist

## Structure

- [ ] Uses SectionNav with client:load
- [ ] First panel is .panel.active, others are .panel
- [ ] Panel ids use panel-{section.id}
- [ ] Sections follow the required order (thesis -> summary)

## Evidence and verification

- [ ] Verified facts are separated from hypotheses
- [ ] Each factual claim has a source link and date
- [ ] Conflicting sources are noted and reconciled
- [ ] Uncertain claims list what would verify them

## Hypotheses and new ideas

- [ ] Hypotheses section is clearly labeled
- [ ] Each hypothesis has rationale and falsification signal
- [ ] New ideas are marked as conjecture

## Visualizations

- [ ] Minimum 3 visuals proposed
- [ ] At least 1 Mermaid diagram included
- [ ] At least 1 C4 or draw-io diagram specified
- [ ] Each visual ties to a claim or model

## Content quality

- [ ] Defines key terms and scope boundaries
- [ ] Includes tradeoffs and failure modes
- [ ] Provides implementation patterns and examples
- [ ] Includes evaluation metrics and measurement plan
- [ ] Ends with actionable takeaways
