# Domain: facts/conventions

This domain establishes project-wide formatting and coding conventions to ensure consistency across HTML, JSX, and MDX files.

## Core Formatting Standards

### Tag Syntax and Bracket Placement
The project enforces a specific rule for tag boundaries to maintain structural consistency:
*   **Closing Bracket Alignment**: For all HTML, MDX, and JSX tags, the closing angle bracket (`>`) must remain on the same line as the tag boundary. Leading brackets on subsequent lines are prohibited.
*   **Reference**: [html_tag_formatting.md](html_tag_formatting.md)

## Key Concepts Overview
*   **HTML/JSX/MDX Consistency**: Uniform application of formatting rules across different markup and component formats.
*   **Bracket Placement**: Strict adherence to same-line closing brackets to prevent fragmented tag definitions.
