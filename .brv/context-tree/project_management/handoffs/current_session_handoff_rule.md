## Raw Concept
**Task:**
Document the handoff file guideline to limit content to the current session.

**Changes:**
- Reinforced that only tasks completed within the current session should appear in future handoffs.
- Confirmed that previously checked items are excluded to avoid redundancy.
- Emphasized keeping handoffs concise and focused on current work.

**Flow:**
Current session completion -> Capture completed tasks -> Exclude previous sessions -> Keep summary concise -> Deliver handoff file

**Timestamp:** 2026-04-07

**Author:** Team Process Guidance

## Narrative
### Structure
The guideline applies whenever handoff files are created so they only reflect work completed during the active session.

### Dependencies
Depends on consistently tracking completed tasks per session so that only new items are included.

### Highlights
Focusing handoffs on current work prevents duplication, reduces noise, and keeps recipients clear on outstanding tasks.

### Rules
Rule 1: Only include tasks completed in the current session.
Rule 2: Do not re-include items that were checked in earlier sessions.
Rule 3: Keep the handoff description brief and strictly centered on current work.

## Facts
- **handoff_scope**: Include only completed tasks from the current session in future handoffs. [convention]
- **handoff_non_duplication**: Do not repeat checked items from previous sessions once they have already been handed off. [convention]
- **handoff_brevity**: Keep handoff notes brief and focused strictly on the work that remains or is newly completed in the current session. [convention]
