## Technical Details

### URL Hash Navigation Flow:

1. User clicks section in SectionNav → `activateSection(id)` called
2. URL hash updated to `#{id}` via `window.history.replaceState()`
3. All panels hidden, target panel (`panel-{id}`) shown with `active` class
4. On mobile (<1024px), panel scrolled into view
5. Browser back/forward triggers `hashchange` → section re-activated
6. Page load with hash triggers inline script → panel activated + SectionNav synced

### Custom Event:

- `section-activated`: Dispatched from inline script to update SectionNav state
- Payload: `{ sectionId: string }`
